## 📌 주제 1: Groq 모델 정리

### 배경
- Groq API에서 `deepseek-r1-distill-llama-70b` 모델이 서비스 종료됨
- 해당 모델을 openclaw.json 설정에서 제거 필요

### 작업 내용
- `openclaw.json`에서 삭제된 Groq 모델 제거
- `reserveTokensFloor: 4000` 옵션 추가
  - **이유**: Groq 소형 모델들의 context limit exceeded 오류 방지
  - 응답 생성을 위한 최소 토큰 공간을 항상 예약

### 핵심 개념
```
reserveTokensFloor: 4000
→ 입력 컨텍스트가 길어도 최소 4,000 토큰은
  항상 응답 생성용으로 남겨둠
→ "컨텍스트 한계 초과" 오류를 사전에 방지
```

### 강의 포인트
- **API 모델 수명주기**: AI 모델도 서비스가 종료된다. 설정 파일에 명시적 버전 관리 필요
- **토큰 예약 전략**: 긴 대화에서 소형 모델이 응답을 못하는 문제를 시스템 레벨에서 해결

## 📌 주제 2: Quartz 퍼블리싱 시스템 구축

### 배경
- 옵시디언 노트를 웹에 공개하고 싶은데, 전체 공개는 부담
- **선택적 공개** 시스템이 필요 (특정 노트만 공개)

### 도구 선택: Quartz
- **Quartz**: 오픈소스 옵시디언 → 정적 웹사이트 변환 도구
- GitHub Pages와 연동하여 무료 호스팅 가능
- `publish: true` frontmatter가 있는 노트만 선택적 공개

### 구축 과정
1. **GitHub 레포 생성**: `keyfream7001/john-brain`
2. **Quartz 설치 및 설정**
3. **동기화 스크립트 작성**: `C:\JohnCodeQ\obsidian-quartz\sync-obsidian.ps1`
4. **GitHub Pages 배포 설정**
5. **옵시디언 가이드 노트 작성**: `600 Specialties/Quartz 퍼블리싱 가이드.md`

### 최종 결과
```
공개 URL: https://keyfream7001.github.io/john-brain/
동기화: sync-obsidian.ps1 실행 → 자동 빌드 → GitHub Pages 게시
공개 조건: frontmatter에 publish: true 있는 노트만
```

### 활용 방법
```yaml
---
publish: true   # 이 줄이 있어야 웹에 공개됨
---
```

### 강의 포인트
- **PKM(개인 지식 관리) + 퍼블리싱**: 지식 관리 도구를 콘텐츠 발행 플랫폼으로 확장
- **선택적 공개 전략**: 개인 노트와 공개 노트를 같은 볼트에서 관리하는 효율적 방법
- **정적 사이트의 장점**: 서버 비용 0원, GitHub Pages 무료 호스팅
