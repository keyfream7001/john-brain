# 1. Executive Summary

## 한 줄 요약

> **바이브코딩 시대, 코딩을 모르는 사용자가 코드의 안전성을 판단할 수 있게 해주는 AI 보안 검증 플랫폼.**

## 핵심 요약

### 문제
바이브코딩 시장이 연 38% 성장하며 전 세계적으로 $4.7B 규모에 도달했다(2025). Cursor, Replit Agent, v0 등으로 **코딩을 모르는 수백만 명이 코드를 생성·설치·배포**하고 있다. 그러나 이들은 자신이 실행하는 코드가 안전한지 판단할 능력이 없다. OS 보안 시스템(Defender, Gatekeeper)은 사용자가 직접 실행한 `npm install`, `pip install`을 신뢰하여 통과시키고, 기존 보안 도구(Snyk, Semgrep, Socket.dev)는 모두 **개발자 전용**으로 설계되어 있다.

Unit 42(Palo Alto Networks) 연구에 따르면, 5개 바이브코딩 도구로 만든 15개 앱에서 **69개의 보안 취약점**이 발견됐다.

### 솔루션
**CodeTrust**는 코드를 업로드하면 AI가 "이 코드가 무엇을 하는지"를 **사람의 언어로 설명**하고, 위험 요소를 찾아 판단을 도와주는 서비스다. 정적 분석(Semgrep + Tree-sitter) + AI 코드 리뷰(Claude API) + 동적 분석(샌드박스)를 결합한 올인원 파이프라인으로, **비개발자가 이해할 수 있는 한국어 보고서**를 생성한다.

### 시장 기회
- **TAM**: 글로벌 애플리케이션 보안 시장 $10.6B (2025, Grand View Research)
- **SAM**: 바이브코더 + 노코더 + 외주 구매자 대상 보안 서비스 ~$800M
- **SOM**: 한국 시장 초기 진입 — 연 ₩30억 (3년 내)

### 비즈니스 모델
B2C SaaS (무료 ~ 월 ₩99,000 4단계 구독) + B2B 플랫폼 연동(크몽, 위시켓 검증 뱃지)

### 자금 소요
- MVP 개발 (3개월): ₩2,000만 ~ ₩3,000만
- Phase 1 운영 (6개월): ₩5,000만
- 총 시드 투자 요청: **₩8,000만 ~ ₩1억**

### 목표
- 6개월 내: MVP 런칭 + 유료 사용자 500명
- 12개월 내: MRR ₩2,000만 + 크몽/위시켓 B2B 계약
- 24개월 내: 일본·영어권 확장, 시리즈 A 준비

# 2. 문제 정의 — 시장의 Pain Point

## 2.1 원점: 실제 경험에서 출발한 문제

2022년, 크몽에서 온라인 견적서 프로그램을 구매했다. 기능은 괜찮았지만 한 가지가 이상했다. **카카오톡 발송 기능**이 내장되어 있었는데, 개발자가 카카오 비즈메시지 API 비용(건당 과금)을 별도로 청구하지 않았다.

나중에 직접 코딩을 배운 후 해당 프로그램의 코드를 분석했다. 결과: **견적서에 입력되는 고객 데이터(이름, 연락처, 금액)가 외부 서버로 전송될 수 있는 구조**였다. API 비용을 안 받은 이유가 설명됐다 — 데이터 자체가 수익이었을 가능성이 높다.

이것은 한 사람의 이야기가 아니다. **코드를 읽을 줄 모르는 모든 사용자**가 매일 직면하는 위험이다.

## 2.2 바이브코딩이 만든 구조적 위험

2025~2026년, 바이브코딩이 폭발적으로 성장했다. Cursor, Replit Agent, Claude Code, v0 등으로 **코딩을 모르는 사람도 앱을 만들 수 있는 시대**가 열렸다. 그러나 역설이 존재한다:

| 할 수 있는 것 | 할 수 없는 것 |
|-------------|-------------|
| AI로 코드를 생성 | 생성된 코드를 읽고 이해 |
| `npm install` 실행 | 패키지의 안전성 판단 |
| 앱을 배포 | 보안 취약점 검증 |

**바이브코더의 전형적 위험 시나리오:**

1. 유튜브에서 "10분만에 SaaS 만들기" 영상을 본다
2. 영상이 시키는 대로 터미널에 명령어를 복붙한다 — `npm install`, `pip install`
3. 에러가 나면 AI한테 물어보고, AI가 시키는 대로 `sudo npm install` 등 실행
4. **뭘 설치했는지, 그게 뭘 하는지 전혀 모른 채 다음 단계로 넘어간다**

## 2.3 OS 보안의 구조적 사각지대

이것이 치명적인 이유: **OS 보안 시스템은 "사용자가 직접 실행한 것"을 신뢰하기 때문이다.**

| OS | 동작 | 결과 |
|----|------|------|
| **Windows Defender** | 사용자가 직접 실행한 `npm install`은 간섭하지 않음 | "사용자가 원한 행동"으로 통과 |
| **macOS Gatekeeper** | App Store 외부 앱은 경고하지만 `pip install`은 검사 안 함 | 터미널 패키지 무검증 |
| **Linux** | `sudo` 입력 시 시스템의 모든 문이 열림 | 무제한 권한 부여 |

## 2.4 클라우드 환경의 착각

RunPod, Google Colab 같은 클라우드 환경은 로컬 PC와 격리된다. 그러나:

- `.env` 파일의 **API 키**(OpenAI, Stripe)가 악성 패키지에 의해 외부 전송 가능
- **SSH 키, GitHub PAT** 탈취 시 연결된 모든 서비스 노출
- AI 에이전트에게 부여한 **MCP 토큰** 유출 가능

격리된 환경이라고 안심하는 순간, **진짜 가치 있는 것(크레덴셜)**은 이미 노출된다.

## 2.5 데이터로 보는 문제의 규모

| 지표 | 수치 | 출처 |
|------|------|------|
| 바이브코딩 시장 규모 | $4.7B (2025) → $12.3B (2027 예상) | Second Talent, CAGR 38% |
| Cursor 기업 가치 | $400M → **$10B** (12개월 내) | Market Clarity (2025) |

| npm 악성 패키지 | 연간 수천 건 탐지, 증가 추세 | Socket.dev, Snyk 보고서 |
| 한국 GPTers 커뮤니티 | **10만+ 회원** | GPTers 공식 |
| 한국 IT 외주 시장 | **연간 수조 원** | 크몽, 위시켓, 숨고 합산 |

# 3. 솔루션 — CodeTrust

## 3.1 제품 개요

**CodeTrust**는 코드나 프로그램을 넣으면, AI가 "이 코드가 뭘 하는지"를 사람 말로 설명해주고, 위험 요소를 찾아 알려주는 서비스다.

안티바이러스와 다르다. 바이러스 시그니처를 찾는 게 아니라, **코드의 행동을 분석하고 설명**하는 것이다.

**출력 예시:**
- "이 코드는 당신의 연락처 데이터를 `api.unknown-server.com`으로 전송합니다."
- "이 npm 패키지는 6개월 전에 메인테이너가 바뀌었고, 새 버전에 난독화된 코드가 추가됐습니다."
- "이 프로그램은 백그라운드에서 포트 8443을 열고 외부 연결을 수신합니다."

## 3.2 포지셔닝 — 에어백 모델

**우리는 "100% 안전합니다"라고 말하지 않는다.** 그건 사기다.

에어백은 100% 생존을 보장하지 않지만, 장착하지 않는 차는 없다.

| ❌ 우리가 하지 않는 약속 | ✅ 우리가 하는 약속 |
|----------------------|-----------------|
| "이 코드는 100% 안전합니다" | "이 코드의 위험도를 분석해서 판단을 도와드립니다" |
| "모든 위협을 막겠습니다" | "알려진 위험 패턴의 80~90%를 탐지해서 경고합니다" |
| 사고 시 책임 전가 | "분석 범위와 한계"를 항상 명시 |

**비즈니스 관점에서도 이 포지셔닝이 정확하다:**
- "100% 보장" → 한 번 뚫리면 신뢰 완전 붕괴 + 법적 책임
- "위험도 분석 + 판단 지원" → 지속 가능한 신뢰 관계

## 3.4 핵심 설계 원칙

1. **설명이 먼저, 점수는 나중** — "위험도 7.2"보다 "이 코드는 당신의 고객 전화번호를 외부로 보냅니다"가 중요
2. **한국어 네이티브** — 보고서, UI, 알림 모두 한국어 (글로벌 확장 시 다국어)
3. **Zero Setup** — 회원가입 → 코드 업로드 → 결과 확인. 설치할 것 없음
4. **솔직한 한계 명시** — 분석 범위와 한계를 항상 표기

# 5. 경쟁 분석

## 5.1 직접 경쟁사 비교

| 서비스 | 가격 | 타겟 | 핵심 기능 | CodeTrust와의 차이 |
|--------|------|------|----------|-------------------|
| **Snyk** | 무료~Enterprise | DevSecOps 팀 | 의존성 취약점, 컨테이너 스캔 | CI/CD 전제, 비개발자 사용 불가, 한국어 없음, 백도어 탐지 불가 |
| **Socket.dev** | 무료/Team $100+/월 | 개발팀 | npm/PyPI 공급망 위험 탐지 | 패키지만 분석 (프로젝트 코드 미지원), 영어 전용 |
| **Semgrep** | OSS/유료 | 개발자 | 정적 코드 패턴 매칭 | 규칙 작성 필요, 개발자 전용, 자연어 설명 없음 |
| **VirusTotal** | 무료/유료 | 일반인/보안팀 | 파일 시그니처 악성코드 탐지 | 새로 짠 악성 코드는 통과, 코드 행동 분석 불가 |
| **npm audit** | 무료 (내장) | npm 사용자 | CVE 체크 | 알려진 취약점만, 공급망 공격 감지 불가 |
| **GitGuardian** | 무료~팀 | 개발팀 | 하드코딩 시크릿 탐지 | 시크릿만 (코드 행동 분석 없음) |

## 5.2 핵심 차별화 포인트

| 차별화 축 | 기존 도구 | **CodeTrust** |
|----------|----------|---------------|
| **사용자** | 개발자 전용 | **비개발자 퍼스트** |
| **보고서** | CVE-2024-1234, CVSS 7.5 | **"이 코드는 고객 전화번호를 외부로 보냅니다"** |
| **분석 범위** | 단일 영역 (의존성 OR 시크릿 OR 패턴) | **올인원: 정적 + AI + 동적 + 공급망** |
| **언어** | 영어 | **한국어 네이티브** |
| **바이브코더 맞춤** | 미고려 | **npm install 전 인터셉트, Cursor 확장** |
| **신뢰 모델** | "취약점 N개 발견" | **"위험도 분석 + 판단 지원" (에어백 모델)** |

## 5.3 경쟁 우위의 지속성 (Moat)

1. **한국어 특화 + 로컬 생태계 연동** — Snyk/Socket.dev의 한국 진출 우선순위 낮음. 크몽/위시켓 연동은 글로벌 도구가 절대 안 함
2. **비개발자 UX 노하우** — 기술 보고서를 비개발자용으로 번역하는 건 단순 번역이 아닌 UX 설계 역량
3. **한국 서비스 특화 탐지 규칙** — 카카오 API, 네이버 클라우드, 토스 결제 패턴 등 글로벌 도구에 없는 규칙
4. **커뮤니티 기반 DB** — 사용자 신고 → 분석 → DB 반영 → 집단 지성 효과

| 영역 | 기술 | 선택 근거 |

| **DB** | Supabase (PostgreSQL) | Auth, Storage, Realtime 내장 |
| **인프라** | AWS (ECS + Lambda) | 샌드박스: ECS, 정적 분석: Lambda 서버리스 |
| **큐** | Redis + Bull | 분석 작업 큐잉, 다수 요청 동시 처리 |

### 핵심 분석 엔진 상세

**정적 분석:**
- Tree-sitter AST 파싱 → 함수 호출 추출 → 위험 함수 DB 매칭
- Semgrep 커스텀 규칙 (외부 통신, env 접근, eval/exec, postinstall 등)
- 자체 데이터 흐름 추적: 민감 데이터 소스 → 싱크(HTTP 전송, 파일 기록) 경로 탐지

**AI 코드 리뷰:**
- 정적 분석 플래그 영역 → LLM 의도 추론 → 비개발자용 한국어 설명 생성
- Haiku로 1차 필터링 → Sonnet으로 2차 정밀 분석 (비용 최적화)
- Few-shot: 정상 코드 vs 악의적 코드 비교 프롬프트

**동적 분석 (Phase 2):**
- Docker + gVisor 격리 환경에서 실행
- mitmproxy: HTTPS 트래픽 프록시
- eBPF/strace: 시스콜 모니터링
- 실행 시간 제한 60초

**패키지 DB:**
- npm 상위 10만개 + PyPI 상위 5만개 프리스캔 → 나머지 온디맨드
- 메인테이너 변경 이력, 타이포스쿼팅 탐지, postinstall 스크립트 분석
- 커뮤니티 리포트 → DB 반영 루프

# 12. 리스크 분석

| 리스크 | 심각도 | 확률 | 대응 방안 |
|--------|--------|------|----------|
| **LLM 할루시네이션** — AI가 없는 위험을 만들어냄 | 🔴 높음 | 중 | 정적 분석 결과를 근거로만 설명 생성. AI 단독 판단 금지. "AI 추론 부분" 명시적 표기 |
| **오탐(False Positive)** — 정상 코드를 위험으로 판단 | 🔴 높음 | 높 | 화이트리스트 DB (카카오 SDK, Firebase 등). 사용자 피드백 루프로 지속 개선 |
| **미탐(False Negative)** — 악성 코드를 놓침 | 🔴 치명 | 중 | "100% 보장" 절대 안 함. 보고서에 "분석 범위와 한계" 항상 명시. 에어백 포지셔닝 |
| **경쟁사 진입** — Snyk/GitHub이 비개발자 타겟 확장 | 🟡 중간 | 중 | 한국어/로컬 생태계 선점. 크몽 연동 등 로컬 경쟁우위. 빠른 실행 |
| **API 비용 급증** — Claude API 과금 폭증 | 🟡 중간 | 중 | Haiku 1차 필터링 + Sonnet 정밀 분석. 캐싱 적극 활용. 오픈소스 LLM 대안 준비 |
| **법적 이슈** — 타인 코드 분석의 저작권 | 🟢 낮음 | 낮 | 보안 분석은 공정 이용 범위. 원본 코드 분석 후 즉시 삭제 정책 |
| **시장 교육 비용** — "보안 검증"의 필요성을 모르는 사용자 | 🟡 중간 | 높 | 콘텐츠 마케팅으로 문제 인식 확산. 크몽 사례 등 구체적 스토리텔링 |
| **인력 확보** — 보안 + AI + 풀스택 인재 부족 | 🟡 중간 | 중 | 바이브코딩으로 소수 정예 개발. 원격 근무로 인력 풀 확대 |

## A. 기술 스택 상세 아키텍처

### 정적 분석 엔진 탐지 대상

| 카테고리 | 탐지 패턴 | 구현 방식 |
|---------|----------|----------|
| 외부 통신 | `fetch`, `axios`, `requests.post`, `urllib` | AST 함수 호출 추출 → URL 분류 |
| 데이터 수집 | `fs.readFile`, `os.environ`, 키 입력 캡처 | Semgrep 규칙 |
| 난독화 | `eval()`, `exec()`, base64 문자열, 동적 import | 패턴 매칭 + AI 추론 |
| 권한 상승 | `sudo`, 레지스트리 수정, cron 등록 | Semgrep 규칙 |
| 크레덴셜 접근 | `.env` 읽기, `os.environ` 순회, AWS 크레덴셜 | 데이터 흐름 추적 |
| 의존성 체인 | package.json, requirements.txt 분석 | 패키지 DB 조회 |
| 인스톨 스크립트 | postinstall, preinstall, setup.py cmdclass | 특별 감시 규칙 |
