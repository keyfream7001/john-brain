# 유튜브 대본: “요청 버튼 한 번으로 앱이 자동 생성되는 시스템 만들기 (10고개 앱빌더)”

## 오프닝 (0:00)
(인트로)
안녕하세요. 오늘은 제가 하루 동안 어떤 걸 만들었는지 보여드릴게요.

주제는 딱 하나입니다.
“관리자가 버튼 한 번 누르면, 앱 프로토타입이 자동으로 만들어져서 링크로 돌아오는 시스템.”

## 문제 제기 (0:20)
요즘 AI로 앱 만들기 콘텐츠 많죠.
근데 대부분은 ‘한 번 만들고 끝’이에요.

저는 반대로,
‘앱을 만드는 과정을 자동화하는 서비스’를 만들고 있었어요.
그게 바로 10고개 앱빌더입니다.

## 오늘의 목표 (0:45)
오늘 목표는 이 플로우를 실제로 완성하는 거였어요.
1) 관리자가 앱 제작 요청을 올린다
2) 웬디가 대기열을 확인한다
3) 자동으로 프로토타입을 만든다
4) 배포 URL을 돌려준다

## 1) 먼저 버그/기반부터 정리 (1:10)
자동화는 기반이 튼튼해야 해요.
그래서 먼저,
- Supabase RLS 권한 이슈
- 관리자 버그리포트 권한 체크
- 프로젝트 저장 버그
- PC 프리뷰 UX(모바일 기본 + 전환 버튼)

이런 걸 정리해서 신뢰성을 올렸습니다.

## 2) 서비스 기능을 붙이기 (2:00)
그 다음은 기능 개발.
오늘 추가한 것들:
- 컬러 피커(브랜드 컬러 설정)
- 테스터 기능(지정하면 무료)
- 결제 후 9단계 온보딩
- 관리자 프로젝트 요청 페이지(목록/상세/상태변경)
- 웬디 작업 요청 버튼

그리고 핵심!
`/api/wendy/pending-requests`

이 API로 웬디가 “대기 중 요청”을 읽어갈 수 있어요.

## 3) 자동화 + 보안은 세트 (3:20)
이런 자동화는 악용되면 바로 사고 나거든요.
그래서 보안 룰도 같이 만들었습니다.
- 금지 키워드 필터링
- 의심스러우면 즉시 중단
- API 응답에 보안 지침 자동 포함

## 4) 결과: 첫 프로토타입 생성/배포 성공 (4:10)
그리고 드디어…
존이 테스트로 요청을 하나 넣었고,
웬디 조수가 프로토타입을 만들고,
배포 링크가 나왔습니다.

배포 URL: https://prototype-promotion.vercel.app

화면은 5개:
- 랜딩
- 로그인
- 프로모션 목록/상세
- 마이페이지

디자인은 와디즈 스타일, 브랜드 컬러는 `#3b3758`.

## 5) 운영 자동화까지 (5:10)
여기서 끝이 아니라,
웬디 상태가 바뀌면 프로젝트 상태도 자동으로 바뀌게 했어요.
- working → in_progress
- completed → completed
- 결과 URL 자동 저장

이제 운영자가 하는 건 ‘관리’뿐이고,
실행은 시스템이 합니다.

## 클로징 (6:00)
오늘 보여드린 건 “AI로 앱을 만든 것”이 아니라,
“앱을 만들게 하는 시스템”을 만든 과정이었어요.

다음 영상에서는
요청이 들어왔을 때 웬디가 어떤 순서로 처리하는지,
아키텍처 그림으로 한 번에 정리해볼게요.

구독/좋아요…는 알아서 해주시고,
궁금한 점은 댓글로 남겨주세요.
