## 1) 고객/일상 지원: NC 신구로점 ‘피자몰(피자몰투고)’ 메뉴 확인 요청 대응
### 배경
- 존이 **NC 신구로점**에서 친구와 식사/메뉴 선택 중.
- 요청: “피자몰에 있는 피자 메뉴를 알려줘 / 메뉴 전부 정리해줘”

### 웬디의 처리 흐름(핵심 전략)
1. **지점/시즌별 메뉴 변동 가능성**을 먼저 명시
2. 웹에서 확인 가능한 **대표 라인업(추정)**을 제시하되, “정확한 전체 목록은 캡처 필요”라고 안내
3. 배달앱(배민/쿠팡이츠) 메뉴 화면 캡처 요청 → “그 화면 기준으로 빠짐없이 목록화 + 가격/추천까지 가능” 제안

### 실제 제공 내용(요약)
- 지점별 편차가 커서 “공식 텍스트 메뉴판”만으로는 **완전한 목록이 어렵다**는 점을 설명
- **확률 높은 피자 라인업 예시**(대표 메뉴) 제시
  - 아메리칸 수프림
  - 치즈퐁듀 쉬림프
  - 불고기 BBQ
  - 베이컨 골드 체다
  - 스테이크 베이컨 웨지 포테이토
  - 하와이안
- 정확한 정리를 위해 **메뉴 화면 캡처 1~2장 요청**

### 커뮤니케이션 이슈
- 존이 “앤디야”라고 부르며 호칭 혼선 발생
- 웬디가 “존, 나 웬디야”로 정정하고 요청을 계속 받는 형태로 수습

## 2) 웹 리서치/검증: ‘피자몰(이랜드잇츠)’ 정보 확인
### 수행한 리서치
- Brave web_search/web_fetch로 피자몰 관련 공식/리뷰성 페이지 탐색
- 이랜드잇츠(브랜드 소개) 페이지에서 **가격/컨셉(뷔페, 시즌 메뉴 등)** 확인 시도

### 발생한 문제와 해결
- Brave Search API 파라미터 검증 오류(422)
  - 원인: ui_lang/search_lang 값이 허용된 enum과 불일치(예: ko, ko-KR 혼용)
  - 대응: ui_lang를 **ko-KR**, search_lang를 **ko**로 조정하며 정상 응답 확보

### 참고로 확인된 내용(요지)
- 공식/소개 페이지는 “뷔페/시즌 메뉴/다양한 메뉴”를 설명하지만,
  - **지점별 실제 피자 종류(전체 리스트)**는 공개되어 있지 않거나 변동 가능
- 결론: “대표 메뉴 안내 + 현장/배달앱 캡처 기반 확정 리스트화”가 최적
