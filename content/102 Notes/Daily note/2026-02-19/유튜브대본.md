### [오프닝 — 0:00~0:30]

**[화면: 유튜브 자동화 파이프라인 다이어그램]**

안녕하세요, 존입니다.

유튜브 영상 자동화하면서 항상 걸리는 문제가 뭔지 아세요?

바로... **TTS 비용**이에요.

영상 하나 만들 때마다 API 비용 나가고, 양이 늘어날수록 청구서가 무서워지죠.

그래서 오늘은 제가 직접 집에 있는 서버 PC로 AI 음성 서버를 만든 과정을 공유할게요.

---

### [문제 제기 — 0:30~1:30]

**[화면: 기존 TTS API 비용 비교 표]**

현재 주요 TTS 서비스들이에요.

ElevenLabs, OpenAI TTS, Google Cloud TTS...

퀄리티는 좋은데 **매달 나가는 비용이 문제**죠.

특히 유튜브 영상이나 콘텐츠를 대량으로 만들면, TTS 비용만 몇 십만 원씩 나올 수 있어요.

그래서 생각한 게 **로컬 AI TTS**예요.

---

### [솔루션 소개 — 1:30~2:30]

**[화면: Qwen3-TTS GitHub 페이지]**

오늘 소개할 모델은 **Qwen3-TTS**예요.

알리바바에서 만든 오픈소스 TTS 모델인데요,

특히 **0.6B, 1.7B**같은 경량 버전이 있어서 집에 있는 게이밍 PC로도 돌릴 수 있어요.

저는 GTX 1060 6GB짜리 서버 PC에서 돌렸는데... 됩니다! 진짜로!

---

### [설치 과정 — 2:30~4:30]

**[화면: 터미널 명령어]**

먼저 Python 가상환경 만들고요.

```bash
python -m venv qwen-tts-env
qwen-tts-env\Scripts\activate
pip install torch transformers
```

그 다음 모델을 로드하는데...

**⚠️ 주의사항 하나!**

0.6B 모델 쓰시면 이런 오류가 납니다.

```
CUDA assert input[0] != 0
```

저도 여기서 엄청 삽질했어요. 결론은?

**1.7B-Base 모델 쓰세요.** GTX 1060에서도 잘 돌아갑니다.

**[화면: 코드 에디터 - 서버 코드]**

API 서버는 FastAPI로 구성했어요. 포트 7860에 띄우고요.

---

### [핵심 꿀팁 — 4:30~6:00]

**[화면: 파형 비교 — 잘린 것 vs 정상]**

자, 여기서 제가 가장 오래 삽질한 부분을 알려드릴게요.

TTS 생성하면 **끝 글자가 잘리는 문제**가 있어요.

"안녕하세요" 하면 "안녕하세"로 나오는 거죠.

처음엔 비율로 자르려고 했는데, 말하는 속도가 문장마다 달라서 이 방법은 못 씁니다.

**[화면: 해결 코드 - 더미 텍스트 + 무음 감지]**

제가 찾은 해법은 이거예요:

**1단계**: 원본 텍스트 뒤에 더미 문장 추가
```python
text = original_text + "...그리고 잠시 쉬어가겠습니다."
```

**2단계**: 음성 파일에서 무음 구간 감지

**3단계**: 무음 직전 지점에서 정확하게 자르기

이렇게 하면 **100% 깔끔하게** 잘립니다.
