# 만원코딩 고객 프로젝트 작업 방식

> 고객 프로젝트 진행 시 계정/서비스 세팅 방법 정리

---

## 🎯 방식 비교

| 방식 | 장점 | 단점 |
|------|------|------|
| **A. 존 계정 개발 → 이전** | 고객 부담 0, 빠름 | 이전 작업 필요 |
| **B. 고객 가입 → 팀 초대** | 이전 불필요, 깔끔 | 고객이 가입해야 함 |
| **C. 하이브리드** | 유연함 | 서비스별 다르게 관리 |

---

## 🅰️ 방식 A: 존 계정으로 개발 → 완성 후 이전

### 흐름
1. 존 계정에서 모든 세팅 & 개발
2. 완성품 확인 → 고객 결제
3. 고객 계정 생성 → 이전

### 이전 작업
- **Supabase**: 새 프로젝트 + 스키마/데이터 복사
- **Vercel**: 고객 계정에 재배포 + 환경변수
- **OAuth**: 리다이렉트 URL 변경
- **도메인**: 새 계정에 연결

### 비용
- Supabase 동시 2개까지 무료
- 그 외 무료

### 추천 상황
- 고객이 개발 완전 모를 때
- 빠른 MVP 필요할 때

---

## 🅱️ 방식 B: 고객 가입 → 팀 초대

### 흐름
1. 고객에게 가입 가이드 전달 (링크 + 스크린샷)
2. 고객이 가입 후 존을 팀원으로 초대
3. 존이 세팅 & 개발
4. 완성 후 존만 팀에서 나가기

### 서비스별 초대 방법

| 서비스 | 초대 방법 | 무료? |
|--------|-----------|-------|
| **Supabase** | Organization → Members → Invite | ✅ |
| **GitHub** | Settings → Collaborators → Add | ✅ |
| **카카오 개발자** | 앱 → 팀 관리 → 팀원 초대 | ✅ |
| **Google Cloud** | IAM → 주 구성원 추가 | ✅ |
| **Firebase** | 프로젝트 설정 → 사용자 및 권한 | ✅ |

### 제한 있는 서비스

| 서비스 | 상황 | 해결책 |
|--------|------|--------|
| **Vercel** | Hobby(무료)는 팀 X | GitHub push로 배포 |
| **PG사** | 사업자 계정, 팀 X | API 키만 전달받기 |

### 추천 상황
- 장기 유지보수 계약
- 고객이 직접 관리 원할 때

---

## 🅲️ 방식 C: 하이브리드 (추천!)

### 서비스별 최적 방식

| 서비스 | 추천 방식 | 이유 |
|--------|-----------|------|
| **GitHub** | 고객 계정 + 초대 | 코드 소유권 명확 |
| **Supabase** | 고객 계정 + 초대 | DB 소유권 중요 |
| **Vercel** | 존 계정 → 이전 | Hobby 팀 제한 |
| **카카오/구글** | 고객 계정 + 초대 | API 키 소유권 |
| **PG사** | 고객 계정 + 키 전달 | 사업자 필수 |

### 고객 가이드 템플릿

```markdown
## 🚀 프로젝트 시작 전 준비사항

### 1. GitHub 가입 (코드 저장소)
- https://github.com 가입
- [존 이메일] 을 Collaborator로 초대

### 2. Supabase 가입 (데이터베이스)  
- https://supabase.com 가입
- Organization 생성 → [존 이메일] 초대

### 3. 카카오 개발자 (로그인 기능)
- https://developers.kakao.com 가입
- 앱 생성 → 팀원에 [존 이메일] 추가

### 4. (선택) 도메인
- 원하는 주소 있으면 알려주세요
- 없으면 무료 주소로 시작 가능!

---
❓ 어려우시면 화면공유로 도와드릴게요!
```

## 💳 결제 관련

### 고객에게 필요한 것 (결제 기능 있을 때)
- 사업자등록증
- 대표자 이메일
- 통장사본 (정산용)

### PG사 세팅
- 토스페이먼츠 / 키움페이 / 카카오페이
- 고객이 가입 → API 키 발급 → 존에게 전달
- 웹훅 URL은 존이 세팅
