# 다한힘 시스템 가이드

> 코드베이스 직접 분석 기반 — 실제 구현된 내용만 기록 (2026-03-13)

## 📋 1. 시스템 개요

**다한힘(Dahanhim)**은 방송/엔터테인먼트 업계의 캐스팅 플랫폼이다.
방송사 스태프가 공고를 올리고, 기획사 매니저가 소속 아티스트를 지원하는 B2B 캐스팅 매칭 서비스.

**기술 스택**
- Frontend: Next.js (App Router, TypeScript)
- Backend: Supabase (PostgreSQL + Realtime + Auth + Storage)
- 스타일: Tailwind CSS

### 핵심 사용자 유형 4가지

| 유형 | DB 필드 | 설명 |
|------|---------|------|
| **매니저** | `user_type = 'manager'` | 기획사 소속, 아티스트 관리, 공고 지원 |
| **스태프** | `user_type = 'staff'` | 방송사 소속, 공고 작성, 아티스트 섭외 |
| **협회관리자** | `role = 'association'` | 협회 공지/투표 관리, 회원사 현황 조회 |
| **슈퍼관리자** | `role = 'admin'` | 전체 시스템 관리, 광고/유저/계약 등 |

> `user_type`은 매니저/스태프 구분, `role`은 권한 등급(user/association/admin)

## 🤝 2. 매니저 ↔ 스태프 상호작용 시스템

### 역할 요약

**매니저 (Manager)**
- 기획사 소속
- `managers` 테이블에 등록, `agencies` 테이블과 연결
- `manager_artists` 테이블로 담당 아티스트 관리
- 공고 발견 → 소속 아티스트로 지원 → 미팅 수락/거절 → 채팅 → 계약

**스태프 (Broadcast Staff)**
- 방송사 소속
- `broadcast_staff` 테이블에 등록, `broadcasters` 테이블과 연결
- 공고 작성 → 지원자 검토 → 미팅 일정 제안 → 채팅 → 계약

### 상호작용 플로우

```
[스태프]                         [매니저]
   │                                │
   ▼                                │
공고 작성 (postings)                │
   │                                │
   │         공고 발견              │
   │◄─────────────────────────────── │
   │                                ▼
   │                      캐스팅 요청 (casting_requests)
   │                      status: pending
   │                                │
   ▼                                │
지원자 목록 검토                    │
   │                                │
   ▼                                │
face_meeting 생성 ──────────────────►│
(미팅 일정 제안)                    │
   │                                ▼
   │                      미팅 수락/거절
   │                      face_meetings.status → completed/cancelled
   │                                │
   ▼                                ▼
채팅으로 세부 조율 (chat_rooms / chat_messages)
   │                                │
   ▼                                ▼
전자계약 (contracts)
```

### 각 단계별 DB 테이블

| 단계 | 테이블 | 핵심 상태값 |
|------|--------|------------|
| 공고 작성 | `postings` | open / closed / cancelled |
| 캐스팅 지원 | `casting_requests` | pending / accepted / rejected / cancelled |
| 미팅 일정 | `face_meetings` | scheduled / completed / cancelled / no_show |
| 참석 응답 | `face_meetings.attendance_status` | pending / confirmed / declined |
| 채팅 | `chat_rooms`, `chat_room_members`, `chat_messages` | — |
| 계약 | `contracts` | draft / sent / signed_a / signed_b / completed / cancelled |

## 📢 3. 공고 시스템 상세

### 공고 작성 주체
- **스태프만** 공고 작성 가능
- 코드: `if (!user || user.type !== 'staff') { alert('방송 스태프만 공고를 작성할 수 있습니다') }`
- 작성 전 **강제시청 광고(MandatoryAdViewer)** 시청 필수

### 공고 상태
| DB 값 | 화면 표시 | 설명 |
|-------|----------|------|
| `open` | 모집중 | 지원 가능 |
| `closed` | 마감 | 지원 불가 |
| `cancelled` | 촬영완료 | 종료 |

### 공고 필드
- 제목 (title), 프로그램명 (program_name)
- 프로그램 유형: ENT(예능) / DRAMA / AD(광고) / MUSIC / NEWS / ETC
- 촬영일 (shoot_date), 촬영 장소 (shoot_location)
- 출연료 범위, 캐스팅 분야, 제외 분야, 성별 조건
- 캐스팅 분야 옵션: `배우, 가수, 코미디언, MC, 모델, 스포츠선수, 방송인, 인플루언서, 기타`

### 자동 만료 처리
- 공고 목록 페이지 진입 시 `GET /api/postings/expire` 자동 호출
- 촬영일이 지난 공고 자동 마감 처리

### 매니저의 공고 보기
- 전체 공고 목록 조회 가능 (본인 것만 X)
- 아티스트가 1명이면 바로 지원, 복수이면 아티스트 선택 모달 표시
- 지원 완료 시 "지원완료" 버튼 표시 (재지원 불가)

### D-day 뱃지
- D-7 이내: 빨간색 (urgent)
- D-8 이상: 파란색
- 촬영 지남: 회색

## 🎤 4. 캐스팅 요청 시스템

### 구조
- 매니저가 아티스트를 공고에 지원 → `casting_requests` 테이블 INSERT
- `posting_id`, `artist_id`, `manager_id`, `request_type = 'apply'`, `status = 'pending'`

### 상태 흐름
```
pending → accepted (스태프 수락 → face_meeting 자동 생성)
         rejected (스태프 거절)
         cancelled (매니저 취소)
```

### face_meeting 자동 생성 트리거
- 스태프가 지원자 목록에서 특정 아티스트 선택 시 → face_meeting 생성
- 경로: `/meetings/[posting_id]/applicants`

### 매니저 본인의 지원 목록
- 경로: `/my-applications`
- `casting_requests` 테이블에서 `manager_id` 필터링

## 💬 5. 채팅 시스템

### 구조
- 1:1 채팅 (캐스팅 연결 기반)
- `chat_rooms.casting_request_id`로 어떤 공고/아티스트 관련인지 추적
- `chat_room_members`로 참여자 관리 (user_id, unread_count, last_read_at)

### 실시간 (Supabase Realtime)
- `postgres_changes` 이벤트 구독
- 테이블: `chat_messages` (INSERT), `chat_rooms` (UPDATE), `chat_room_members` (UPDATE)
- 메시지 전송은 `/api/chat/send` API route 사용 (service role key로 RLS 우회)

### 첨부파일
- 최대 **100MB** (system_settings의 `chat_max_file_size_mb` 값)
- 허용 타입: image, document, video (`chat_allowed_types`)
- 업로드: `/api/chat/upload`
- 메시지 형식:
  - `[이미지] 파일명\nURL`
  - `[동영상] 파일명\nURL`
  - `[파일] 파일명\nURL`

### 채팅방 목록 표시
- 스태프 뷰: 엔터사명 + 매니저이름 + 아티스트 배지
- 매니저 뷰: 매니저 이름 + 공고 제목

### Optimistic Update
- 메시지 전송 시 임시 ID로 즉시 화면에 표시
- 실패 시 "전송 실패 / 재시도" 표시

## 🏛 6. 협회관리자 역할

### 권한 조건
```typescript
const isAssociationAdmin = user?.role === 'association' || user?.role === 'admin'
```

### 접근 경로
- `/association` — 협회게시판

### 가능한 기능들

#### 소속 매니저 목록 조회
- `users` 테이블에서 `user_type = 'manager'` 필터링
- 표시 정보: 이름, 이메일, 인증 여부, 가입일
- 협회관리자만 열람 가능 (일반 매니저/스태프는 볼 수 없음)

#### 공지사항 관리 (association_notices)
- 작성/수정/삭제 권한: `role IN ('admin', 'association')`
- 읽기 권한: 전체 로그인 유저
- 필드: 제목, 내용, 중요 여부 (important: boolean)
- 아코디언 UI로 내용 펼치기/접기

#### 투표 시스템
- 테이블: `votes`, `vote_options`, `vote_responses`
- 투표 생성/관리: 협회관리자만
- 투표 참여: 전체 로그인 유저 (1인 1투표, UNIQUE 제약)
- 익명 투표 지원 (`is_anonymous: boolean`)
- 투표 기간 설정 (`starts_at`, `ends_at`)
- 결과 실시간 집계 (응답 카운트 표시)
- 투표 만들기: `/association/votes`

## 👑 7. 슈퍼관리자 기능

### 권한 조건
- `role = 'admin'`
- 접근 경로: `/admin` (일반 `/login`과 별도로 `/admin-login`으로 추정)

### 대시보드 (`/admin`)
KPI 카드:
- 총 회원수, 승인 대기 광고, 집행중 광고, 전체 계약
- 매니저 수, 스태프 수, 협회관리자 수

### 유저 관리 (`/admin/users`)
- 전체 유저 목록 조회 (탭: 전체/매니저/스태프/협회관리자)
- 표시: 이름, 이메일, user_type, role, 인증여부, 가입일
- API: `GET /api/admin/users`, `GET/PATCH /api/admin/users/[id]`

### 광고 관리 (`/admin/ads`) — 4개 탭
1. **이미지 배너** (`/api/admin/ad-banners`)
   - 4종: manager-vertical, manager-horizontal, staff-vertical, staff-horizontal
   - 크기: 300×250, 728×90
   - URL 변경만 가능 (이미지 직접 업로드 아님)

2. **동영상 광고** (`/api/admin/ad-videos`)
   - YouTube URL 또는 직접 비디오 URL
   - 노출 대상: 전체/매니저/스태프
   - 상태: pending/approved/active/inactive/rejected/expired
   - 강제 시청 / 스킵 허용 설정

3. **강제시청 광고** (`/api/admin/mandatory-ad`)
   - 공고 작성 전 전체화면 강제 시청
   - 설정: 영상 URL, 스킵 허용 여부, 강제 시청 시간(초)
   - system_settings 테이블에 저장

4. **광고 승인** (`/api/admin/ads`)
   - pending 상태 광고 승인/거절

### 계약 관리 (`/admin/contracts`)
- `contracts` 테이블 조회
- API: `GET /api/admin/contracts`
- 상태: draft / sent / signed_a / signed_b / completed / cancelled

### 시스템 설정 (`/admin/settings`)
- `system_settings` 테이블 CRUD
- API: `GET/PATCH /api/admin/settings`
- 기본 설정 항목:
  - `chat_max_file_size_mb`: 100
  - `chat_allowed_types`: image,document,video
  - `ad_slot_count`: 5
  - `ad_display_seconds`: 10

### 통계 (`/api/admin/stats`)
- KPI 데이터: totalUsers, pendingAds, activeAds, totalContracts, managers, staff, association
- 최근 광고 목록 (recentAds)

### 폴백 동영상 (`/admin/fallback-videos`)
- 페이지 존재, 상세 기능 미확인

## 📊 9. 전체 데이터 플로우 다이어그램

```
[스태프] ──► 공고작성(postings: open)
               │
               ▼
[매니저] ◄── 공고 목록 조회 (전체 open 공고 노출)
               │
               ▼ 지원하기 클릭
[매니저] ──► casting_requests INSERT (status: pending)
               │
               ▼
[스태프] ◄── 지원자 목록 (/meetings/[postingId]/applicants)
               │
               ▼ 선택 → face_meeting 생성
               face_meetings (status: scheduled)
               │
               ▼
[매니저] ◄── 미팅 알림 수신
               │
               ▼ 수락/거절
[매니저] ──► face_meetings UPDATE (status: completed/cancelled)
               │ 수락 시
               ▼
[알림서버] ──► 스태프에게 알림
               │
               ▼
[양측] ──► 채팅방 생성 (chat_rooms)
               │
               ▼
[양측] ──► 계약 (contracts)
```

- `GET /api/casting-requests` — 목록 조회 (posting_id, manager_id, status 필터)

- `GET /api/artists/my` — 내 담당 아티스트

- `POST /api/profile/upload-avatar` — 아바타 업로드

**투표**
- `POST /api/votes/respond` — 투표 참여

**관리자 (admin only)**
